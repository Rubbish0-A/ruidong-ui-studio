# 睿动智能体 UI —— 跨风格共通元原则

> 这份文档定义睿动**所有**智能体 UI 的底线。无论加载哪套具体风格（`styles/<id>/`），下列原则都不能违反。  
> 各风格自己的个性约束写在该风格的 `PHILOSOPHY.md` 里，不在本文件重复。

---

## 1. Token 纪律（Token Discipline）

**代码里只用 token 名，不写具体值。**

- ✅ `bg-accent`、`text-ink`、`var(--accent-1)`
- ❌ `bg-[#5e6ad2]`、`color: #ff2e97`、`rgba(255,46,151,0.3)` 直接写死

理由：hex 写死意味着切换子主题时每一行都要改；token 引用只需换一个变量文件。

如果某个视觉值当前主题的 token 里没有，先提议是否将它加入 `TOKENS.md`，说明理由，而不是直接写 arbitrary 值。

---

## 2. 字体底线（Typography Floor）

- ❌ **禁止使用 Inter** — 来自真实用户反馈，Inter 气质与睿动"高级感"目标不符
- ✅ 每套风格必须在 `TOKENS.md` / `fonts.html` 中声明自己的字体对，并提供完整的中文 fallback
- ✅ 中文 fallback 至少包含：`"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif`
- ❌ 不得在主题声明之外使用 `system-ui` 作为首选字体（系统默认字体无法跨设备保证视觉一致性）

---

## 3. 四层层级系统（Depth System）

每个风格可以自由调整视觉表现，但层级名称和语义必须就是这 4 层，不允许增减：

| 层级 | 语义 | 毛玻璃 |
|---|---|---|
| **Base** | 页面背景 | ❌ |
| **Surface** | 卡片、内容区 | ❌ |
| **Raised** | 悬浮卡片、下拉菜单 | ❌ |
| **Overlay** | 弹窗、命令面板、顶栏、Toast | ✅ 仅此层允许 |

"毛玻璃只允许在 Overlay 层"是所有风格都必须遵守的底线，不因风格个性而例外。

---

## 4. 可访问性底线（Accessibility Floor）

- ✅ 所有动效必须响应 `prefers-reduced-motion: reduce`（降级为无动画或瞬切）
- ✅ 正文文字对比度不得低于 WCAG AA（4.5:1）
- ❌ 不得用颜色作为传递信息的**唯一**手段（状态必须同时有图标或文字）

---

## 5. 布局自由原则（Layout Freedom）

**这个 skill 只管配色和字体，不锁定布局。**

- 页面结构（有无侧边栏、几列 Grid、卡片大小）由业务逻辑决定
- 不得强行要求特定布局
- 具体风格的 `PHILOSOPHY.md` 可以"推荐"布局模式，但不得将其作为强制条件

---

## 6. 信息密度原则（Density Principle）

页面按密度分三级，具体风格可调视觉表现，但不得混乱：

| 类型 | 密度 | 典型场景 |
|---|---|---|
| 展示型 | 低 | 欢迎页、空状态、Hero 区 |
| 工作台 | 中 | 主功能页、对话 |
| 配置型 | 中高 | 设置页、管理列表 |

同一页面内密度必须一致。

---

## 7. 与 Claude 协作时的硬约束

当 Claude 在按某套睿动风格写代码时：

1. **Token-only**：引用 token 名称而非具体值（hex / rgb / rgba）
2. **主题锁定**：生成代码前先检查项目根是否有 `ruidong-ui.config.json`；有则严格按 `style` + `subtheme` 字段生成，不得自行切换
3. **声明式变更**：如需新增视觉值，先提议将它加入当前主题的 `TOKENS.md` 并说明原因
4. **风格内部一致**：同一项目不得混用两套不同的 style 或 subtheme
5. **尊重风格边界**：`smartsolu-linear` 和 `bold-vivid` 是两套完全不同气质，不能把一套的 token 用在另一套里

---

## 8. 风格族附加原则（Style-Family Principles）

> 以下规则只在对应风格族内有效，不是全局底线。

### `smartsolu-linear` 风格族

- 品牌气质：**专业 · 精致 · 克制**（"冷静、聪明、轻盈的专业工作台"）
- 全产品只允许 **1 个主强调色**，用于主按钮、选中态、关键链接
- 画面 85%+ 由中性色承担
- 功能色（成功 / 警告 / 错误）必须降饱和
- 渐变是点缀，不是铺底
- 动效快速微妙：**150–300ms**，不超过 400ms
- 禁止：大面积纯色铺底、多强调色共存、毛玻璃用在主内容区

### `bold-vivid` 风格族

- 品牌气质：**强识别度 · 反平庸 · 主题化**（"每个界面都应有独特的视觉记忆点"）
- 允许 2–4 个强调色，但必须有明确的层级分工（primary / secondary / tertiary accent）
- 背景可使用主题化渐变和纹理层
- 动效可以是编排式入场序列（stagger，0.8s 以内为宜）
- 字重使用极端对比（100–200 vs 800–900）而非安全的 400/600
- 禁止：Inter 字体 · 无主题感的紫蓝渐变 · 完全无入场动效 · 平白无纹理底色
