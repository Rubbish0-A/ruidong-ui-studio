# smartsolu-linear · 设计哲学

## 叙事主线

**冷静、聪明、轻盈的专业工作台。**

所有设计决策都要先通过这句话的过滤器。看起来高级但不属于这个气质的设计 —— 不要。

## 风格主次（三源融合，不平均混合）

| 学 | 用于 |
|---|---|
| **主骨架学 Notion** | 信息组织、阅读舒适度、卡片布局、留白 |
| **细节质感学 Linear** | 精致阴影、微妙渐变、状态过渡、深色侧边栏 |
| **浮层交互学 Raycast** | 毛玻璃、悬浮层、轻盈感 |

**规则**：主内容区稳定实心（Notion），表面质感精致（Linear），**只有浮层才用玻璃效果**（Raycast）。

## 五条设计原则

1. **界面先保证秩序和呼吸感，再讨论装饰细节**
2. **深度只通过有限的层级系统表达，不靠特效堆砌**
3. **色彩只服务于重点、状态和品牌识别**
4. **所有交互反馈都应快速、克制、连续**
5. **排版优先建立信息优先级，其次才是风格气质**

## 绝不做的事

- ❌ 不为显高级而滥用玻璃、发光、渐变
- ❌ 不让任何组件脱离系统单独炫技
- ❌ 不把精致做成炫技（阴影/玻璃/渐变一多就过度）
- ❌ 不把克制做成单薄（太白太灰太轻会失去存在感）
- ❌ 不把简洁做成空（没有视觉焦点用户不知道看哪里）
- ❌ 不使用 Inter 字体
- ❌ 不在主内容区使用毛玻璃
- ❌ 不叠加多种强调色

## 四层层级系统

| 层级 | 用途 | 背景 | 阴影 | 描边 | 毛玻璃 |
|---|---|---|---|---|---|
| **Base** | 页面背景 | `bg-canvas` (#f7f8fc) | 无 | 无 | 否 |
| **Surface** | 卡片、内容区 | `bg-white` | `shadow-surface`（双层） | `border border-edge` (#e6e9f0) | 否 |
| **Raised** | 悬浮卡片、下拉菜单 | `bg-white` | `shadow-float`（三层） | `border border-edge-strong` (#cdd1dc) | 否 |
| **Overlay** | 弹窗、命令面板、顶栏、Toast | `bg-white/80` | `shadow-overlay` | `border border-edge/60` | **是** |

**深色侧边栏例外**：采用 Linear 深色面板（`bg-sidebar` #0f1011 + 半透明白色边框），与主内容区形成强对比。这是 SmartSolu 风格的标志之一。

## 信息密度分级

| 页面类型 | 密度 | 特征 | 典型页面 |
|---|---|---|---|
| 展示型 | 低 | 大留白、弱边框、弱分隔、强焦点 | 空状态、欢迎页 |
| 工作台 | 中 | 明确分区、卡片组织、操作区清晰 | 主功能页、对话 |
| 配置型 | 中高 | 分组排版、紧凑但可读 | 设置页、知识库管理 |

## 状态规则

| 状态 | 视觉表现 |
|---|---|
| default | 实心白底 + 极淡描边 + 双层柔和阴影 |
| hover | 描边变色（accent-soft） + 阴影加深 + 轻微上浮（-translate-y-0.5） |
| active / pressed | scale-[0.98] + 阴影收缩 |
| focus | ring-2 ring-accent/10 + border-accent |
| selected | border-accent + bg-accent-soft + ring-1 |
| disabled | opacity-50 + cursor-not-allowed |
| loading | 骨架屏或柔和脉冲（animate-pulse），不用粗暴 spinner |

## 动效规则

| 类型 | 时长 | 用途 |
|---|---|---|
| hover 反馈 | 150ms ease | 按钮/卡片/链接悬停 |
| 面板展开 | 200ms ease | 下拉菜单、历史面板、Tab 切换 |
| 弹窗出入 | 300ms `cubic-bezier(0.2,0,0,1)` | Modal、命令面板 |

动效不是"看见动画"，而是"状态切换显得顺手"。
